## Architecture

Next.js 16 App Router site for AItomation Academy (Codex education for non-technical professionals). Deployed on Vercel with server-side API routes.

### Key architectural decisions

- **Vercel deployment**: Pages are statically generated at build time. API routes (`app/api/`) run as serverless functions. No `output: 'export'` — removed to enable API routes.
- **Single-page composition**: `app/page.tsx` composes all sections in order. Each section is a self-contained component in `app/sections/`.
- **Two component directories**: `app/components/` holds page-specific components (navbar, lead magnet form). `components/` at the root holds app-wide concerns (PostHog provider, analytics events, structured data).
- **CSS variables design system**: Colors are defined as HSL CSS variables in `app/globals.css` and consumed via Tailwind config (`tailwind.config.js`). Uses the shadcn/ui variable naming convention but does not use shadcn components.
- **Site config centralized**: All site metadata, URLs, social links, and keywords live in `lib/site.ts`. The canonical URL is `https://www.aitomationacademy.com` (with www). Structured data (JSON-LD) is generated from this config in `lib/schema.ts`.
- **Analytics**: PostHog initialized via `instrumentation-client.ts` (Next.js 15.3+ approach). Only runs in production. Custom event tracking with section-aware CTA attribution via `data-section` attributes. Requires `NEXT_PUBLIC_POSTHOG_KEY` and `NEXT_PUBLIC_POSTHOG_HOST` env vars.
- **Email integration**: Brevo API via `/api/subscribe` route. Lead magnet forms create contacts in Brevo list 6 (Website Lead Magnet) and send welcome email (template 1). Newsletter form uses list 5 and template 12. PDF is delivered via email, not instant download.
- **Blog**: Articles live in `content/blog/{slug}.tsx`. Metadata, TOC, related articles, and lead capture CTA are auto-generated by the page template at `app/blog/[slug]/page.tsx`.

### Environment variables

- `NEXT_PUBLIC_POSTHOG_KEY` — PostHog project API key
- `NEXT_PUBLIC_POSTHOG_HOST` — PostHog instance host
- `BREVO_API_KEY` — Brevo v3 API key (server-side only, starts with `xkeysib-`)

## When adding new pages

Every new page must be wired into these files or SEO will break:

1. **`app/sitemap.ts`** — Add the route to the `staticRoutes` array. Set appropriate `priority` (0.7-0.9) and `changeFrequency`.
2. **Canonical URL** — Add `alternates: { canonical: \`${siteConfig.url}/your-route\` }` to the page's `metadata` export. Without this, the root canonical (`/`) propagates incorrectly.
3. **`public/llms-full.txt`** — Add the URL to the "Canonical pages" section. For blog posts, add to the "Published articles" list. Always use `www.aitomationacademy.com` (with www).
4. **`public/llms.txt`** — Update if the new page represents a new topic area.
5. **Internal links** — Every page should have at least 2 incoming internal links from other pages. Orphaned pages hurt SEO.

### When adding blog posts

Use the `aitomation-blog-writer` skill (`.Codex/skills/aitomation-blog-writer/SKILL.md`). It handles the full pipeline. Key rules:

- **Title: max 50 characters** — the site appends " — AItomation Academy" (~24 chars)
- **Description: max 155 characters**
- **Verify external URLs** before including them — broken links hurt SEO
- **Use `/skool-redirect`** for all community links (not direct Skool URLs)
- **Use `www.aitomationacademy.com`** for all canonical references
- After publishing: add to `lib/blog.ts`, `public/llms-full.txt`, and run `npm run build`

## Links and URLs

- **Canonical domain**: `https://www.aitomationacademy.com` (with www) — defined in `lib/site.ts`
- **Skool community**: Always link via `/skool-redirect` page (fires PostHog tracking event before redirect). Never link directly to `skool.com`.
- **Contact email**: `contact@aitomationacademy.com` — use `siteConfig.email` from `lib/site.ts`, don't hardcode
- **YouTube**: Use `siteConfig.social.youtube` from `lib/site.ts`
- **Old URLs**: `/privacy-policy` → `/privacy`, `/tos` → `/terms` redirects are in `next.config.js`

## CTA tracking

All CTA sections have `data-section` attributes for PostHog event attribution. When adding new sections with CTAs:

- Add `data-section="section_name"` to the section element
- The `cta_click` event in `components/analytics-events.tsx` automatically picks up the section from the nearest `[data-section]` ancestor
